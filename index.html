HTML

  • The head element contains information about the webpage.
  • The head element is not visable to the user and contains valuable information for the browser.
  • The body element represents the visible content shown to the user.
  • The body element should have each piece of content marked in the appropriate tags for easy code readability.

CSS

  • A margin indicates how much space we want around the outside of a element.
  • A padding indicates how much space we want around the content inside of an element.
  • A border is between the margin and padding modifiers of an element and can be modified in different ways.
  • A class allows you to modify any elements that you assign that class to.

GIT

  • git status: checks what branch we are currently on
  • git checkout -b branchname: creates a new branch and switches to it

JavaScript

  • JavaScript is a dynamic language with variable types. Variables in JavaScript are not directly associated with any particular value type.
  • Variables are containers that store values.
  • Variables need to be given a name.
  • JavaScript is case sensitive. This means that "myVariable" is not the same as "myvariable".
  • Variables may hold values that have different data types.
  • Some examples of data types that can be assigned to variables are: String, Number, Boolean, Array, and Object.
  • After assigning a value to a variable, you can change it later in the code.
  • String: a sequence of text that requires a single or douoble quote mark at the begining and end to identify it.
  • Number: numbers don't have quotes around them.
  • Boolean: is a true/false value and the words "true" and "flase" are special keywords that don't need quote marks.
  • Array: is a structure that allows you to store multiple values in a single reference.
  • Object: can be anything. Everything in JavaScript is an object and can be stored in a variable.
  • Operators: allow you to use a mathmatical symbol to produce a result based on two values or variables.
  • Conditionals: Are code stuctures used to test if an expression returns true or not. A common form of this is the "if...else" statment.
  • Functions: are a way of packaging functionality that you wish to reuse. It is possible to define a body of code as a function that exicutes when you call the function name in your code. This is a good alternative to repeatedly writing the same code
  • Arguments: are bits of data that a function often needs to do it's job. Arguments go inside the paranthesis, separated by commas if there are more than one argument.
  • Event Handlers: are code structures that listen foe activity in the browser. An example of an obvious event handler is the "click event" which i sfired by the browser when you click on something with your mouse.
  • Null: is a special value in JavaScript that refers to the absense of a value.
